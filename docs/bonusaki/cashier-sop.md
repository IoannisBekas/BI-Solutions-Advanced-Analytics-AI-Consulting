# Bonusaki Cashier SOP

Use this during the cafe pilot staff training.

## Goal

Cashier validates each Bonusaki reward before giving the customer the prize.
This prevents duplicate use, screenshots, and copied QR codes from becoming
uncontrolled cost.

## Standard redemption flow

1. Customer shows the Bonusaki reward screen or wallet-style pass.
2. Cashier checks that the reward text matches an active cafe reward.
3. Cashier opens the validation page or admin tool.
4. Cashier enters or scans the public reward code.
5. Cashier clicks **Validate** and confirms status:
   - valid: fulfill the reward, then click **Redeem reward**
   - already redeemed: politely decline and call manager if needed
   - invalid: politely decline
   - expired: decline unless manager overrides outside Bonusaki
6. Cashier confirms the page shows redeemed after the reward is given.
7. Cashier records unusual issues in the support log.

## Cashier PIN rules

- keep the PIN private from customers
- do not print the PIN on public materials
- change the PIN when staff changes or abuse is suspected
- only the manager and trained cashiers should know it

## What not to accept

- plain screenshots without validation
- edited images
- codes not found in the validation tool
- rewards from another merchant or campaign
- rewards that are already redeemed
- QR screenshots that never produced a valid server reward code

## Customer wording

Valid reward:

> Great, this reward is valid. I will redeem it now.

Already used:

> This reward is showing as already redeemed. I cannot use it again, but I can
> ask the manager to review it if you think this is wrong.

Invalid code:

> I cannot validate this code in the system, so I cannot redeem it.

## End-of-day check

- manager checks issued vs redeemed counts
- manager reports any suspicious clusters to Provider
- Provider reviews event and redemption logs if abuse is suspected

## Emergency pause

Pause the campaign if:

- a QR code appears on social media or outside the cafe
- customers redeem without purchasing when purchase is required
- repeated invalid/duplicate attempts occur
- staff PIN may have leaked

Contact Provider and request QR batch or campaign pause.
