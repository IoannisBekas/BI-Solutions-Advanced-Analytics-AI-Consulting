Methodology

What the atlas is designed to answer

This independent country-level briefing consolidates open public datasets into one starting point for risk, hazard, readiness, loss, finance, donor, and project-signal analysis.

  • What is the current composite disaster and humanitarian crisis risk profile?
  • Which hazards are flagged as high or medium?
  • What loss, displacement, readiness, and climate-vulnerability indicators are available?
  • How much international DRR-related ODA is reported for the country?
  • Which donors and public project-finance signals are visible?
  • How does DRR-related ODA compare per person or against GDP?
  • Are there current GDACS alerts for operational context?

Source-linked metrics make it easier to move from a country screen to the original statistical, budget, project, and finance systems when detailed verification is needed.

Build documentation

How this atlas was built

The atlas is a static GitHub Pages application designed for public research use. It combines a repeatable Node.js data pipeline with a lightweight D3 front end, so the public site can load quickly without a live database server.

Architecture

  • Front end: `index.html`, `styles.css`, and `app.js` render the atlas, charts, filters, country profiles, public exports, and dataset Q&A.
  • Data pipeline: `scripts/download-data.mjs` downloads, normalizes, and joins public DRR datasets by ISO3 country code.
  • Processed outputs: `data/processed/dashboard-data.json` powers the public atlas. CSV exports are generated in-browser from the public atlas dataset.
  • Validation: `scripts/validate-data.mjs` checks source counts, country coverage, schema expectations, missing values, and warning conditions before refreshed data can be committed.
  • Automation: `.github/workflows/refresh-data.yml` runs the public data refresh weekly and blocks publication when strict validation warnings are present.
  • Hosting: GitHub Pages serves the static atlas from the public repository.

Design and interaction

The visual system uses an Economist-inspired editorial style: strong typography, restrained panels, black rules, a focused color palette, direct data labels, explanatory notes, and source-linked metric cards. The experience is organized around six research actions: scan global risk, ask a dataset question, inspect one country, compare two countries, review finance signals, and check severity-finance alignment.

Severity and reporting-gap logic

The severity layer combines public deaths, displacement, direct-loss, loss/GDP, hazard, and risk signals. Observed severity is a 0-100 public impact index derived from the latest available public indicators where available.

Compound hazard pressure is a 0-100 proxy based on ThinkHazard high and medium hazard counts plus INFORM hazard, vulnerability, and coping-capacity dimensions. The atlas describes this as a proxy for compounding or cascading pressure; it does not prove causal event chains.

Reporting-gap signals are labelled low, medium, or high when modeled hazard or vulnerability is high but public impact indicators are sparse or missing. These are screening signals for follow-up verification, not factual conclusions that a country under-reported losses.

World Bank WDI population and GDP are used as denominators for per-person and GDP-share indicators. GDACS current alerts are used as live operational context and are not counted as historical disaster-loss records.

Accessibility and mobile hardening

The atlas includes skip links, semantic landmarks, visible labels, chart summaries, accessible data tables behind visual charts, focus-visible states, reduced-motion CSS, keyboard-reachable documentation tables, color contrast checks, and mobile overflow checks. The latest local and live axe-core runs returned zero automated violations across the tested atlas and documentation views.

Source scope

Downloadable and reference source layers

The public atlas uses sources that can be downloaded or queried programmatically. Additional source systems are documented as reference and future research layers for discovery.

Source group Atlas use Interpretation note
INFORM, WorldRiskIndex, ThinkHazard Risk, exposure, vulnerability, and hazard screening Mixed model assumptions and source years.
OECD CRS Reported DRR-related ODA by donor, recipient, sector, and flow International support layer for donor and recipient screening. Preliminary aggregate ODA releases are context, not replacements for detailed CRS breakdowns.
World Bank and GCF Public project finance signals and examples Project amounts are used as discovery signals and linked back to source records.
OCHA FTS Humanitarian response funding context Response funding is shown as a separate humanitarian context layer.

The public site uses the public datasets listed in this source scope. Atlas records link back to source publishers where possible.

Accessibility

Accessibility status

Implemented features include a skip link, semantic landmarks, visible labels, chart summaries, accessible chart data tables, keyboard-reachable table scroll regions, focus-visible styles, reduced-motion CSS, axe-core checks, and mobile overflow checks.

Manual keyboard and screen-reader checks are part of the release QA workflow for major updates.

Terms and privacy

Terms

The public atlas is provided for research, country screening, and briefing support. Source links are included for due diligence and follow-up analysis.

The static public site does not require an account. It uses public datasets listed in the source scope and links back to publishers where possible.
